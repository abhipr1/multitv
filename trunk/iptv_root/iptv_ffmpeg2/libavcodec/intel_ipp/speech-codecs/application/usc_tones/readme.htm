Overview

The USC Tone detector sample is a command line based application that performs /ANS tone detection. The TD sample processes input PCM file either narrow band 16bit 8000 Hz or wideband 16bit 16000 Hz and creates one output text file. The output text file contains the textual name of the detected tone. If tone in input file aren’t detected than outfile is empty. The /ANS tone is 2100 Hz tone with phase reversals at 450 ms.

The tone detection algorithm complies with ITU G.168-2002 specifications.

Running the Software

For information on building the sample, see the readme file in the ipp_samples\speech-codecs directory.

In order to run the executable created by this sample,

be sure that the LD_LIBRARY_PATH environment variable includes the directories of the .so files from the Intel ® IPP installation. This can be accomplished by entering the following at the command prompt:   for examples 

export LD_LIBRARY_PATH=/opt/intel/ipp/5.3/ia32/sharedlib:$LD_LIBRARY_PATH

or by running the shell script ippvars32.sh from the Intel ® IPP installation (located in /opt/intel/ipp/5.3/ia32/tools/env/ by default).

./usc_tones -type ANS [options] <inPCMFile> <outTextFile> 

The option list: 


  -fXXX               PCM frequency in kHz of the input file, where XXX equals 8000 or 16000.

 -n                       output additional information  (optional).

  -o <logFile>      write to <logFile> file log information (optional).
<inPCMFile> - input audio PCM file, 8000 Hz  or 16000 Hz

<outTextFile> - output text file.
