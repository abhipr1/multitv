Overview

The USC Echo Canceller sample is a command line based application that performs echo cancellation. The EC sample processes two input files and creates one output PCM files either narrow band 16bit 8000 Hz or wideband 16bit 16000 Hz. The receive-in and send-in samples are processed to perform echo path parameters estimation and estimated echo is subtracted from the send-in samples and "echo-free" PCM data is written to send-out output file.

The sample implements three algorithms:

1.       Fullband algorithm  - processing in time domain,

2.       Subband algorithm – processing in the frequency domain,

3.       Subband-Fast algorithm – fast processing in the frequency domain. The Subband-Fast algorithm introduces 6ms delay for 8kHz and 3 ms delay for 16kHz into the output signal.

The USC Echo Canceller software module is suitable for building an echo cancellation device compliant with ITU G.168-200 2 and G.167 specifications.

Running the Software

For information on building the sample, see the readme file in the ipp_samples\speech-codecs directory.

In order to run the executable created by this sample,

be sure that the LD_LIBRARY_PATH environment variable includes the directories of the .so files from the Intel ® IPP installation. This can be accomplished by entering the following at the command prompt:   for examples 

export LD_LIBRARY_PATH=/opt/intel/ipp/5.3/ia32/sharedlib:$LD_LIBRARY_PATH

or by running the shell script ippvars32.sh from the Intel ® IPP installation (located in /opt/intel/ipp/5.3/ia32/tools/env/ by default).

To run the USC EC sample, type: 

./usc_ec [options] <r-inPCMFile> <s-inPCMFile> <s-outPCMFile>

Option list: 

     [-f  <frequency >] - sample frequency in KHz: 8 or 16. Default: 8.

     [-a <algorithm>] - specify an algorithm: fb (fullband), sb (subband) or sbf (subband-fast). Default: <algorithm> = sb.

     [-ad <mode>] - set adaptation mode: "full" or "lite". "full" is a default value. "lite" mode requires significantly low cpu resources but may show worse adaptation characteristics.

     [-ah <mode>] - Set the anti-howling device mode:1 - on (cos shift up. Exclude sb mode). 0 - off. 0 as default.

     [-l <num>] - echo tail length in ms in range [1, 200].

     [-delay <val>]   Set the delay in send-in PCM file, integer number of milliseconds. The leading end of send-in PCM will be cut off prior to echo cancel. Default: no delay.

     [-s  <rep>] - repeater, how many times input files shall be processed without resetting EC,  default: one time. 

     [-c <csv filename>] - Print out performance info line to specified file (optional).

     [-o <logFile>] - write log information to the file  <logFile> .

     [-list] - print out the modes list.
     [-sys] - print out system information.
     [-nlp <num>] 0- Set NLP off.1 as default.1- NLP type actual for all modes,2- NLP type actual for sbf only.

     [-cng <num>] 0-Set CNG off. 1 as default. Active when NLP is on only.1- CNG type actual for all modes.

                            2- CNG type actual for sbf only (actual with -nlp type 2 only).

<r-inPCMFile> - input file contains receive-in 16-bit PCM signal.

<s-inPCMFile> - input file contains send-in 16-bit PCM signal.

<s-outPCMFile> - output file contains send-out 16-bit PCM signal.

 

Note:  if repeat is specified (-s option) the last result only will be written into the output file.
