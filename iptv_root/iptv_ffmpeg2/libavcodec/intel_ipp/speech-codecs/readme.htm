Overview

This package contains the echo canceller sample (./application/usc_ec), noise reduction and voice activity detector sample (./application/usc_nr), speech codec sample (./application/usc_speech_codec) and the tone detector sample (./application/usc_tones).  

The speech codec sample supports encoding and decoding compliant to the ITU-T G.711/I/II, G.722, G.722.1, G.723.1/A, G.726/A, G.728G//I/H, G.729/A/B/D/E/I/C/C+ recommendations, ETSI GSM 06.90-06.94, 06.10-06.12, 06.31-06.32 recommendations, RFC3951.

The tone detection algorithm complies with ITU G.168-2002 specifications.

The echo cancellation sample complies to the ITU-T G.168-2000 and G.167 recommendations. 

The samples have been implemented using the Intel ® Integrated Performance Primitives (Intel ® IPP) Speech Coding functions. They are command line based applications which process data given by one or more input PCM files and store the result into output bitstreams or PCM files. Both narrow band (8000 Hz)  and wide band (16000 KHz) are supported.

For more information refer to the specific sample readme.htm file.

Installation

Extract all files in l_ipp-sample_*. tgz to a desired folder. Please make sure to preserve the directory structure. The files for the USC Speech Codec and USC Echo Cancellation sample will be found in the ./ipp_samples/speech-codecs folder.

How to Build Source Code

  • Set up your build environment by creating an environment variable named IPPROOT that points to the root directory of your Intel ® IPP installation. For example: /opt/intel/IPP/5. 3 /ia32/
    • To build for an IA-32 system:
      • Run build _32 .sh.
    • To build for an Intel ® Xeon ® Processor with Intel ® Extended Memory 64 Technology system:
      • Run build _ em64t.sh.
    • To build for an Intel ® Itanium ® processor based system:
      • Run build _ 64.sh.
    • To build for an IXP4xx product line system with MontaVista* Linux* 3.0 CEE LE:
      • Run buildiwmmxt_le.sh
    • To build for an IXP4xx product line system with MontaVista* Linux* 3.1 PRO BE:
      • Run buildxscale_be.sh

The easiest way to build all application is to simply run buil.sh located under directory ./ipp_samples/speech-codecs if the target system is based on Intel® Pentium® Processor system. The build*.sh will build the USC speech codec library and four samples, namely the unified echo canceller and the unified speech codec, noise reduction and tone detector. If build is successful all relevant .obj files and .a files are created and stored under directory ./ipp_samples/speech-codecs/bin/ia32/obj/* and  ./ipp_samples/speech-codecs/bin/ia32/lib/* respectively. The .exe files will be placed in ./ipp_samples/speech-codecs/bin/ia32.

The USC library and samples can be built separately for desired platform using build*.sh batch files located in specific folders:

  1. ./ipp_samples/speech-codecs/codec/speech/build*.sh - to build the USC library  
  2. ./ipp_samples/speech-codecs/application/usc_speech_codec/build*.sh - to build USC Speech Codec sample. The USC library must be built prior to the USC speech codec separate build.
  3.  ./ipp_samples/speech-codecs/application/usc_ec/build*.sh - to build USC Echo Canceller sample.
  4.  ./ipp_samples/speech-codecs/application/usc_nr/build*.sh - to build the Noise Reduction and Voice Activity Detector sample.
  5.  ./ipp_samples/speech-codecs/application/usc_tones/build*.sh - to build the Tone Detector sample.

Note, with default settings it will build the sample with Intel C++ compiler. To change defaults see the Makefile for a brief description of the available variables and additional options.

Echo canceller sample (./application/usc_ec) doesn't support IXPP4XX product line system s.

Running the Software

Read the specific sample readme.htm file to learn how to build and run the sample (to locate see the table above).   

After building, be sure that the LD_LIBRARY_PATH environment variable includes the directories of the .so files from the Intel ® IPP installation. This can be accomplished by entering the following at the command prompt:   for examples 

export IPPROOT=/opt/intel/ipp/5.3/ia32

export LD_LIBRARY_PATH=$IPPROOT/sharedlib:$LD_LIBRARY_PATH

or by running the shell script ippvars32.sh from the Intel ® IPP installation (located in /opt/intel/ipp/5.3/ia32/tools/env/ by default).
